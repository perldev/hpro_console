Creating System

Для создания своей Polog-системы вам будет достаточно акаунта Google или вы можете заполнить нашу форму регистрации. После регистрации мы вам выделяни одну базу знаний для хранения ваших фактов и кода. Вы сразу же ее можете использовать через наше REST API

Security and Embedding

Если вы планируете использовать вашу пролог-систему посредством API или JavaScript Widget-a, вам нужно будет указать секретный и публичные ключи, которые будут использоваться при взаимодействие через API или JavaScript Widget, более подробно смотрите в разделе Managing.

Limitation

В стартовом пакете вам доступно 64 Мегабайта оперативной памяти в нашей облачной системе. Вы можете увеличить кол-во хранимых фактов и доступной памяти путем смены пакета. Сейчас в системе доступны три пакет : "Стартовый","Средний", "Промышленный"

Import your knowledge

Вы можете импортировать ваши данные в систему двумя способами 1) Написав соответствующий код на прологе, и взаимодействуя с ним посредством АПИ импортировать ваши данные 2) Загрузить ваши данные на сервер в соответствующеим csv, json формате Второй способ будет гораздо быстрее.

Protocol

Our api let you work with your prolog code in our system from your application via REST-interface

For example you have this code in codeide.com:

        
parent("FirstHumen", pam).

Authorization

At first you need load your namspace “my_expert_system”, send http request

http://codeide.com:8313/prolog/auth/my_expert_system

Permissions to your namespace is avalible only from ip that you have gave us before :

Making a request

Now, in order to get result for prolog request like this: 

?- predecessor(X, Y)
You need to send such http-request : 
curl -d "params=[{\"name\":\"X\"}, {\"name\":\"Y\"}]" http://codeide.com:8313/prolog/create/predecessor
curl -d "params=[{\"name\":\"X\"}, {\"atom\":\"bob\"}]" http://codeide.com:8313/prolog/create/predecessor
curl -d "params=[\"FirstHumen\", {\"name\":\"Y\"}]" http://codeide.com:8313/prolog/create/predecessor

POST "params" contain json array of arguments for aim "predecessor",

If you want use unbound variables X or Y

you must set it like a json object {"name":"X"}, where X is name of variable, which you want use in prolog

for atoms use object like this {"atom":"bob"}

in other cases lists, strings and numbers use constants

If everything is good, you will receive response json 

{"status":"ok","session":"12esf32453"} 
or
{"status":"fail","description":"some description of mistake"}

Checking a result

Our kernel start process your request

For working with thise process you have three commands

curl     http://codeide.com:8313/prolog/process/12esf32453 # for checking results
curl     http://codeide.com:8313/prolog/next/12esf32453 # for find next solution of prolog work
curl     http://codeide.com:8313/prolog/finish/12esf32453 # for finish process of working

First request will return to you json with bounded variables of your request

{"status":true, result:[ {"X": {"atom": "pat" }} , {"Y": {"atom":"jim"}} ] } # for first example 
{"status":true, result:[ {"X": {"atom": "tom" } }] } # for second example
{"status":true, result:[{"Y": {"atom": "pam" }   }] } # for third example

If aim was not reached any solution the response will be : 

{"status":false, result:[ ] }

After you are able ask Prolog to find another solution by sending

curl     http://codeide.com:8313/prolog/next/12esf32453

or you can finish work

curl     http://codeide.com:8313/prolog/finish/12esf32453

Session is created per each reqest and it will be deleted after finishing request

As you noticed, all work is asynchronous

You do not need load namespace after each aim request, you can do it only once

Prolog Documentation

:-

Для объявления предиката используется следующий синтаксис. 

fact(X, Y):- X =Y.

Где fact -название предиката, а в скобках список аргументов. Факт это предикат без тела, можно считать предикат с телом из одной цели true.

,

;

Для составления сложных правил, используется предикат конъюнкции “,” и дизъюнкции “;”.

fact(X, Y, Z):- X =:= 1, Y =:= 2, Z < Y +X.

в данном примере поочередно будут проверяться все цели, и запрос будет удачным, только если все цели будут успешно достигнуты.

fact(X, Y, Z):- X =:=1, Y =:=2; Z <10.

а в этом случае запрос будет успешным либо в случае, если X будет равным 1 и

Y равным 2 или только Z будет меньшим 10.

. - точка служит для обозначения конца предиката или правила. Каждое объявление предиката или факта заканчивается точкой

true - оператор всегда возвращает всегда true

false - оператор возвращает всегда false

! - отсечение решений, после этого оператора пролог не рассматривает возможные решения которые могут быть до него.

[] - список.

Список в прологе представляется в виде головы и хвоста. [Head | Tail] = List, в данном выражение мы получим первый элемент списка List в переменную Head, и список всех оставшихся элементов в переменной Tail, в случае если List не список, то выражение вернет false. Так же возможны следующие варианты этого выражения

[Head1,Head2, Head3| Tail] = List, в этом случае в переменных Head1, Head2, Head3 будут первые три элемента списка List, если List содержит меньшее количество элементов, то выражение вернет false. Так же пролог поддерживает вложенные списки.

is - вычисление арифметического выражения, не стоит путать с сопоставлением

X is 6 +7, запишит в X 13.

+ арифметическая операция сложения

* арифметическая операция умножения

/ арифметическая операция деления

- арифметическая операция вычитания

mod - остаток от деления

Вне контекста оператора is, данные операторы могут быть использованы как структуры.

<,>,>=,=<, =:=, =\= - операторы арифметического сравнения

*Внимание, так как все данные хранятся в строках, то данная реализация пролога в арифметических выражениях будет строку пытаться преобразовать в число, если строка содержит числовое значение.

assert(X) - добавления фактов и правил

asserta(X)

assertz(X) добавления фактов и правил в начало списка

retract(X)

чтобы добавить правило нужно заключить его в дополнительные скобки, пример

assertz( ( max(X,Y,Res) :- X>Y,!,Res is X; Res is Y ) ).

retract удаляет факты и правила

abolish (X) - удаляет все факты и правила соответствующие терму X, где X - терм вида

name/5, name - наименования функтора, 5 арность - всегда возвращает true.

call(X) - вызывает правило в X

not(X) - возвращает true, если аргумент или выражение в скобках возвращает false. - не стандарт !!!

год, месяц, день, час, минуты и записать в Accum, необходимо для работы со временем.

Предикаты добавления связей

assertz(SourceFact, ForeignFact, Body ) - добавления связи между фактом SourceFact и фактом ForeignFact в виде правила Body.

Где SourceFact, ForeignFact - атомы наименования фактов.

Body - либо atom наименования правила, которое уже существует в системе, либо объявление правила, заключенного в круглые скобки. Связь обозначает, что при добавление посредством предиката assert факта SourceFact, будет вызвано правило Body. Подрозумевается что правило Body связывает факты SourceFact и ForeignFact.

Сейчас не могут существовать факты с одинаковым именем и разной арностью в виртуальной памяти.

Сейчас говорит no если просиходит таймаут апи HBAse … т.е. как-будто бы не существует факта, хотя он может существовать. (нужно сделать чтобы выводило другое сообщение)

meta(fact_name, param, X) - предикат выводит информацию о весе факта fact_name или правила, количестве фактов данного типа в системе, кол-во запросов к этому факту(не реализовано), param - наименование интересующего параметра, может быть

count - кол-во фактов в системе

requests - кол-во запросов к факту

weight - вес этого факта в системе,

X - переменная в которую будет записано значение параметра системы, если переменная конкретизирована, то значение будет проверено согласно обычной логике

localtime( In ) - унифицирует в In текущую дату в формате "Jul 17 2012 08:56"

date_diff( FirstDate, SecondDate, Type, Acum) - вычисляет разницу во времени между датами которые содержут термы FirstDate, SecondDate, должны быть строками вида

"Jul 17 2012 08:56", Type - hour|second|day|minute атом указывает в каких единицах возвращать разницу во времени, Acum - пеменная которая будет унифицирована

вычисленным значением. Предикат может генерировать исключительные ситуации
