Use prolog in your application

Our api let you work with your prolog code in our syste from your application via REST -interface

For example you have such code in new.deepmemo.com:

        
parent("FirstHumen", pam).
parent(pam, bob).
parent(tom, bob).
parent(tom, liz).
parent(bob, ann).
parent(bob, pat).
parent(pat, jim).

predecessor(X, Y) :- parent(X, Y).
predecessor(X, Y) :-
	parent(X, Z),
	predecessor(Z, Y).

Now in order to got result for request like this 

?- predecessor(X, Y)
You need sent request like this 
curl -d "params=[{\"name\":\"X\"}, {\"name\":\"Y\"}  ]"   http://new.deepmemo.com:8313/prolog/create/predecessor
curl -d "params=[{\"name\":\"X\"}, {\"atom\":\"bob\"}  ]"   http://new.deepmemo.com:8313/prolog/create/predecessor
curl -d "params=[\"FirstHumen\", {\"name\":\"Y\"}  ]"   http://new.deepmemo.com:8313/prolog/create/predecessor

POST "params" contain json array of arguments for aim "predecessor",

If you want use unbound variables X and Y

you must set it like json object {"name":"X"}, where X it's common name of prolog variable

for attoms use object like this {"atom":"bob"}

in other cases lists, strings and numbers use constants

If everything is good, you will receive response json like this 

{"status":"ok","session":"12esf32453"} 
or
{"status":"fail","description":"some description of mistake"}

Our kernel start process your request

For working with thise process you have three commands

curl     http://new.deepmemo.com:8313/prolog/process/12esf32453 # for checking results
curl     http://new.deepmemo.com:8313/prolog/next/12esf32453 # for find next solution of prolog work
curl     http://new.deepmemo.com:8313/prolog/finish/12esf32453 # for finish process of working

First request return you Json with bounded variables of your request

{"X": {"atom": "pat" } , "Y": {"atom":"jim"} } # for first example 
{"X": {"atom": "tom" }   } # for second example
{"Y": {"atom": "pam" }   } # for third example

After you can ask Prolog to find another solution by sending

curl     http://new.deepmemo.com:8313/prolog/next/12esf32453

or you can finish work

curl     http://new.deepmemo.com:8313/prolog/finish/12esf32453

Session is created per each reqest and it will be deleted after finishing request
